# Платформа управления соревнованиями

## Описание

Веб-приложение на Flask для управления спортивными и образовательными соревнованиями. Платформа предоставляет функциональность для организаторов соревнований и участников с разделением ролей, аутентификацией и управлением результатами.

## Основные возможности

### Для организаторов:
- Регистрация и вход в систему
- Создание и управление соревнованиями
- Просмотр детальной информации о соревнованиях
- Просмотр команд и участников
- Управление результатами соревнований
- Личный кабинет с статистикой

### Для участников:
- Регистрация и вход в систему
- Просмотр доступных соревнований
- Регистрация на соревнования
- Просмотр личных результатов
- Личный кабинет с рейтингом

## Требования
- Основной исполняемый файл: `app.py`.
- Конфигурация: `config.py`.
- Зависимости: `requirements.txt`.
- Шаблоны: папка `templates/`.
- Статические файлы (CSS): папка `static/`.

Требования
- Python 3.10+ (подходит и 3.11)
- PostgreSQL (или другой совместимый сервер, если вы адаптируете `config.py`)

Установка (Windows PowerShell)
```powershell
# Создать виртуальное окружение
python -m venv venv

# Активировать (PowerShell)
.\venv\Scripts\Activate.ps1

# Установить зависимости
pip install -r requirements.txt
```

Конфигурация
- По умолчанию приложение загружает настройки из класса `Config` в файле `config.py`.
- В `config.py` указаны значения по умолчанию:
  - `SECRET_KEY` — ключ Flask; при деплое замените на безопасный секрет.
  - `DB_CONFIG` — словарь с параметрами подключения к базе данных (host, database, user, password, port).
- Настройки по безопасности: **не храните реальные пароли в репозитории**. Для продакшна рекомендуется изменить `config.py`, чтобы брать значения из переменных окружения.

Пример быстрых правок (рекомендуется)
- Откройте `config.py` и замените значения в `DB_CONFIG` на свои либо реализуйте чтение из `os.environ`.

Запуск приложения
- Локально (после активации venv и установки зависимостей):
```powershell
python app.py
```

- После запуска приложение по умолчанию слушает `http://127.0.0.1:5000`.

Структура проекта
- `app.py` — основной Flask-приложение; содержит маршруты для входа/регистрации, панели организатора и участника.
- `config.py` — настройки приложения.
- `requirements.txt` — список зависимостей (`Flask`, `psycopg2-binary`, `bcrypt`).
- `static/` — статические файлы (например, `style.css`).

### Папка `templates/` — HTML-шаблоны:

**Основные шаблоны:**
- `base.html` — базовый шаблон с навигацией и общей структурой
- `index.html` — главная страница приложения
- `login.html` — страница входа в систему
- `register.html` — страница регистрации пользователей

**Панель организатора (`templates/organizer/`):**
- `dashboard.html` — главная панель организатора со статистикой
- `competitions.html` — список всех соревнований организатора
- `add_competition.html` — форма создания нового соревнования
- `competition_detail.html` — детальная информация о соревновании
- `competition_results.html` — управление результатами соревнования

**Панель участника (`templates/participant/`):**
- `dashboard.html` — личный кабинет участника с его данными
- `competitions.html` — каталог доступных соревнований
- `register_competition.html` — форма для регистрации на соревнование
- `results.html` — просмотр личных результатов участника

База данных
- Приложение использует PostgreSQL через `psycopg2`. Схема БД должна содержать таблицы: `Организатор`, `Участник`, `Соревнование`, `Команда`, `Результаты` и т.д., как это предполагается в `app.py`.
- Если у вас нет готовой БД, создайте базу и таблицы согласно использованию в `app.py`.

Безопасность
- Не публикуйте `SECRET_KEY` и пароли к БД в публичных репозиториях.
- Для продакшн-среды отключите `debug=True` в `app.run()` и используйте WSGI-сервер (gunicorn/uwsgi) за обратным прокси.

Дополнительно
- Логика хеширования паролей использует `bcrypt` — это хорошая практика.
- Проверьте и обновите валидацию e-mail/паролей при необходимости.

Как внести изменения и запустить локально
```powershell
# Создать и активировать venv
python -m venv venv; .\venv\Scripts\Activate.ps1
pip install -r requirements.txt

# Отредактировать config.py (при необходимости)
python app.py
```
